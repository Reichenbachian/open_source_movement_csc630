git cheatsheet

Resources Assignments git cheatsheet
  • git clone <repository>: Clones a repository from a remote source in a new directory — the command can be thought as a combination of both git init and git branch [default_branch].
  • git init
  • git status: Displays differences between the index file and the current HEAD, between the working tree and the index file, and paths in the working tree that aren't tracked by Git. Alternatively phrased, it displays what a git commit command would change if run immediately, followed by what would change if files were staged before the commit. Running git diff -s gives less verbose output. Running git diff -v gives very verbose output.
  • git add <filename>
  • git rm <filename>
  • git reset <filename>
  • git commit
  • git diff
  • git log
  • git checkout: Change the workspace (i.e. the current files in the directory) to those located at the specified commit. In essence, moves "you" (a.k.a. HEAD) to the commit. If you add the -b flag, it will create a new branch with the specified name: git checkout -b <new_branch_name>.
  • git branch <branchname>
  • git remote
  • git push : Moves anything that has been committed to the local repository to the remote repository on github. After typing git push, write the name of the remote, typically origin, then the branch you want to commit to.
  • git merge <branchname>
  • git fetch
  • git pull <remote_repo> <remote_branch>
